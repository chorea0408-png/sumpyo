# 숨표

> 예배 준비는 보이게, 내 시간에는 숨표를.

**🔗 라이브 데모:** https://chorea0408-png.github.io/sumpyo/
(모바일 사파리/크롬에서 열고 "홈 화면에 추가"하면 앱처럼 실행됩니다)

3개의 찬양팀(중장년·청소년·청년)을 병행하는 직장인 인도자가 카톡·메모·구글시트에 흩어진
준비 업무를 **한 화면에서 파악**하고, **지금 해야 할 한 가지**를 놓치지 않게 돕는
개인용 주간 운영 대시보드 MVP.

**발표용 한 문장** — "Praise Hub가 팀에게 콘티와 악보를 공유하는 서비스라면,
숨표는 세 개의 찬양팀을 병행하는 개인 인도자가 이번 주의 병목과 다음 행동을 파악하는 서비스입니다."

## 실행

```bash
npm install
npm run dev      # http://localhost:5174
npm run build    # 타입체크 + 프로덕션 빌드
npm run deploy   # 빌드 후 gh-pages 브랜치로 GitHub Pages 배포
```

## 배포 (GitHub Pages)

- 호스팅: GitHub Pages (`gh-pages` 브랜치), 저장소 `chorea0408-png/sumpyo`
- 프로덕션 base 경로는 `/sumpyo/` (Vite `base`), 로컬 개발은 `/`
- 코드를 고친 뒤 `npm run deploy` 한 번이면 라이브 사이트에 반영됩니다.

## 핵심 동작

- **가장 먼저 할 일** — 마감 시각 오름차순 → 체크리스트 순서로 자동 선정
  (마감 초과 → 오늘 → 내일 → 이후가 자연스럽게 보장됨)
- **팀별 진행률** — 완료 토글에 따라 자동 계산, 팀별 저채도 색 구분 (세이지/살구/미스트)
- **빠른 추가** — 이동 중 생각난 일을 10초 안에 기록 (내용·팀·마감일)
- **주간 현황 복사** — 카톡에 붙여넣는 텍스트 요약 (Solo Utility First 전략의 접점)
- **localStorage 영구 저장** — 새로고침해도 유지, 푸터의 '데모 데이터 초기화'로 리셋

## 데모 데이터 설계

시드는 **오늘 기준 상대 날짜**로 생성된다. 어느 요일에 열어도 같은 이야기가 보인다:

- 확인이 필요한 일 1건 (어젯밤 놓친 청소년부 라인업)
- 오늘 마감 3건 (중장년 장비 17:00 · 청소년 전체 공지 18:00 · 청년 콘티 21:00)
- 나머지는 앞으로의 일정 (주간 리듬: 콘티 → 악보·라인업 → 공지 → 연습 → 예배)

## 언어 원칙

업무가 밀렸다는 죄책감을 주지 않는다. "지연" 대신 **"확인이 필요해요"**,
게임식 점수 대신 **"이번 주도 세 번의 예배를 잘 준비했어요"**.
